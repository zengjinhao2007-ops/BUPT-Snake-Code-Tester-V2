# BUPT北邮——贪吃蛇 OJ 本地测试器

快速使用：点dist文件夹exe文件

简单说明：这是一个 Windows 版的 北邮大一计导贪吃蛇大作业 本地测试器，用于调试“贪吃蛇 OJ 交互式题目”的 C 语言程序。

特色：将你的程序可视化。

说明：程序的超时检测仅为参考，不一定代表最终PTA结果，因为不知道计时方式，可能因为CPU型号不同等因素影响。
如果超时，可以在“高级设置”中将时间调高来测试你的程序。

## 主要功能

- 自动检测本机 `gcc`，没有检测到时在中文 UI 中提示安装 MinGW-w64 或 MSYS2。
- 编译并显示 gcc 报错、警告和运行时 `stderr`。
- 模拟 OJ 交互：初始地图和 `N`、每轮读取方向与移动前分数、返回新食物坐标 / `20 20` / `100 100`。
- 可视化 20x20 地图，显示墙、空地、蛇头、蛇身、食物和障碍物。
- 地图外侧显示 `0..19` 的行号和列号，便于按题目坐标数格子。
- 显示评测器内部变量：步数、分数、蛇长、蛇头、食物、方向、是否增长、结束原因等。
- 显示 OJ 交互日志，便于对照程序输出。
- 内置多组固定 seed 测试用例，覆盖样例、撞墙、障碍绕行、自然增长、自撞压力和 OJ 风格随机数据。
- 支持运行当前用例、批量评测、查看每一步地图快照、播放/暂停/调速、导出日志、保存/加载代码。
- 主界面提供显式双选模式控件，可在 `Debug调试` 和 `跑分评分` 之间切换。
- 默认限制与题目要求保持一致：时间 `1000ms`、内存 `64MB`、代码长度 `32KB`、栈 `8192KB`，不再设置步数上限。
- 程序启动和正常退出时会清空 `work` 临时目录，避免分享工具时泄露粘贴过的源码。
- 跑分评分模式也保留逐步地图快照，可查看图形、变量、快照表和播放；若步数极多，会保留开头和末尾关键段以保持界面流畅。真实判题、分数和最终结果不受影响。

## 使用方式

1. 双击 `SnakeOJTester.exe`。
2. 在左侧文本框粘贴你的 C 语言完整程序。
3. 点击“编译”查看 gcc 信息。
4. 点击“运行当前用例”调试单个地图，或点击“批量评测”运行全部内置用例。
5. 在“评测结果”“OJ 交互”“地图快照”“程序错误输出”标签页查看详细信息。

## 模式说明

- `Debug模式`：保留完整调试能力，运行当前用例或批量运行全部内置用例，重点用于定位交互、地图、增长和最终输出问题。
- `跑分模式`：点击“开始跑分(10组)”后，可在“固定组（当前10组）”和“随机组（每次跑分重新生成）”之间切换。每个用例运行 1 次，按文档公式 `总分 = Σ(原始分 / (log2(N) + 1))` 计算加权总分，然后按基础分规则折算，忽略排名奖励分。
- 每次点击“运行当前用例”“批量评测”或“开始跑分(10组)”都会重新编译当前文本框中的代码，并在独立临时目录运行新的 exe。
- 跑分结果会显示本次代码指纹、运行时间和实际运行的 exe 路径。修改代码后，旧结果会标记为“上一次测评结果”，需要重新点击测评按钮才会更新。
- 跑分评分模式会按顺序评测 10 个独立样例，并显示每个样例的实际用时；同时为每个用例保留最多 2500 个逐步地图快照。

基础分规则：

- 加权总分 >= 500：50 分
- 400 <= 加权总分 < 500：45 分
- 300 <= 加权总分 < 400：40 分
- 200 <= 加权总分 < 300：35 分
- 100 <= 加权总分 < 200：30 分
- 加权总分 < 100：0 分

## 判定说明

- “通过”：交互格式、最终地图和得分均正确。
- “答案错误”：分数、最终地图或最终得分与评测器状态不一致。
- “输出格式错误”：方向不是 `W/A/S/D`，或分数不是整数，或最终地图格式不对。
- “超时”：通常是没有 `fflush(stdout)`、程序死循环、等待了错误输入，或收到 `100 100` 后没有输出最终结果。默认 1000ms 后即判超时，但测试器会继续观察到最多 3000ms 并显示诊断用时；若仍未完成，可在“高级设置”中调高时间限制后重测以查看实际分数。
- “内存超限”：学生程序运行时私有内存超过默认 `64MB`。
- “运行错误”：学生程序异常退出或退出码非 0。

## 默认限制

- 时间限制：`1000ms`
- 内存限制：`64MB`
- 代码长度限制：`32KB`
- 栈限制：`8192KB`
- 步数限制：无

代码长度会在编译前按 UTF-8 字节数检查。栈限制通过 MinGW 链接参数设置。运行时按总时间和私有内存监控学生程序。超过时间限制会立即判为超时，但默认最多继续观察到 3000ms 以显示诊断用时；超过诊断上限仍未结束会强制停止该用例，并提示在“高级设置”中调高时间限制后重测以查看实际分数。

性能说明：无步数上限后，优秀策略可能在 1000ms 内产生大量交互。跑分模式默认按每一步保留图形快照，并设置快照上限；超过上限时会保留开头和末尾关键段，判题结果不受影响。

## 随机规则

学校 OJ 的隐藏地图和随机种子无法从题面完全还原。本测试器采用固定 seed 的 C 风格线性同余随机数，在 `1..18` 范围内挑选空地生成食物，尽量贴近交互机制，重点帮助发现协议、状态维护、增长、自撞和最终输出问题。

## 绿色版卸载

关闭程序后，直接删除整个 `SnakeOJTester` 文件夹即可。程序正常退出时会自动清空 `work` 临时目录。
